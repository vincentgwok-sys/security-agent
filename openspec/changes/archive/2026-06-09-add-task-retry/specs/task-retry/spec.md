## 新增需求

### 需求: 中断和失败任务支持一键重试

系统必须允许用户对状态为 `INTERRUPTED` 或 `FAILED` 的任务发起重试，使用原任务的连接信息和 Skill 列表创建新任务并自动启动检测。

#### 场景: 从任务列表重试中断任务

- **当** 用户在任务列表页查看一个 `INTERRUPTED` 任务并点击"重试"
- **那么** 系统必须获取原任务的 SSH 连接信息和 skillIds，调用 `POST /api/tasks` 创建新任务，并在请求中传递 `parentTaskId`
- **且** 新任务必须使用最新的 Skill 版本和规则进行检测

#### 场景: 从任务列表重试失败任务

- **当** 用户在任务列表页查看一个 `FAILED` 任务并点击"重试"
- **那么** 系统同样创建新任务并启动检测

#### 场景: 已完成或运行中任务不显示重试按钮

- **当** 用户在任务列表页查看一个 `COMPLETED` 或 `RUNNING` 的任务
- **那么** 操作列禁止显示"重试"按钮

#### 场景: 重试任务记录父任务 ID

- **当** 重试创建的新任务生成后
- **那么** 新任务的 `parentTaskId` 必须指向原任务的 `taskId`，且 JSON 持久化文件中包含该字段
